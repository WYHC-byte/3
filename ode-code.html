First steps / Quick Start

Quick Start Guide

ode-code CLI (TyloAI-CLI-Coding-Agent) is an intelligent terminal partner. It can read, create, or modify local files while executing commands you know. It requires a one-time global installation.

Installation

Ensure you have Node.js installed, then run:

Terminal
npm install -g @tyloai/ode-code@latest

Login & Initialization

Once installed, start the agent in any project directory:

Terminal
ode code

On your first run, you will be prompted to log in via browser. Afterward, simply navigate to your project folder and run `ode code` to start coding.

CLI Pricing

The CLI consumes credits from your main TyloAI web account. Usage is calculated based on input tokens (context reading) and output tokens (code generation).

Mode Cost Factor Description
Flash Mode 1x Standard token rates. Most economical for quick fixes.

High Mode ~10x - 20x Deep planning and extended reasoning. Highest cost for complex architectural tasks.
Deep Thinking (Tab) 2x Enabling deep thinking doubles the base cost of any request.
Work Modes / Overview

Three Working Modes

Different tasks require different levels of cognitive load. ode-code offers three distinct modes to adapt to your needs.

You can switch modes at any time during a session by typing /mode.

Work Modes / Flash Mode

Flash Mode

The Quick Shooter. Flash Mode is optimized for low-latency, single-turn tasks. It is the default mode when you start the CLI.

When to use Flash Mode:

  • Fixing a simple syntax error or typo.
  • Generating a small utility function.
  • Writing comments or documentation for a single file.
  • Quick queries about the codebase (e.g., "Where is the auth logic defined?").

Characteristics:

Context Window Focused (Current file + user selection)
Reasoning Fast, direct (Model: ode-7-turbo)
Iteration None (Single Shot)
Work Modes / Medium Mode

Medium Mode

The Balanced Iterator. Medium Mode introduces an Agentic Loop. It doesn't just write code; it runs it, checks for errors, and fixes itself.

How it works:

  1. Draft: The AI writes the initial code based on your request.
  2. Verify: It attempts to compile or lint the file (if configured).
  3. Self-Correct: If an error is detected, it reads the error log and applies a fix automatically.
  4. Finalize: It presents the final solution to you only after it passes internal checks.

Best For:

  • Unit test generation (Write test -> Run -> Fix -> Pass).
  • Refactoring components that might break dependencies.
  • Features spanning 2-3 related files.
Work Modes / High Mode

High Mode

The Architect. High Mode is our most advanced agentic workflow. It uses a "Plan-Execute-Review" architecture designed for large-scale changes.

The 3-Step Process:

1. PLANNING PHASE
   - Scans the entire relevant project tree.
   - Generates a markdown plan outlining every step.
   - Asks for user approval before writing a single line of code.

2. EXECUTION PHASE
   - Executes the plan step-by-step.
   - Maintains a "Global Memory" of changes to prevent circular edits.
   - Handles massive context by chunking tasks.

3. REVIEW PHASE
   - Summarizes all changes.
   - Runs integration tests.

Use Cases:

Use High Mode for migrations (e.g., "Convert this whole folder from JS to TypeScript"), scaffolding new microservices, or implementing complex algorithms.

Core Capabilities / Real Code Editing

True Code Editing Capability

ode-code isn't just a chat bot; it's a file system agent. It can manipulate your project structure directly.

Capabilities

  • Create/Delete/Modify: Full CRUD access to project files.
  • Diff View: Before applying any change, ode-code presents a unified diff (colored add/remove lines) so you can review exactly what will happen.
  • Undo Support: Made a mistake? The system supports undo operations to revert changes instantly.
Core Capabilities / Smart Workflow

Smart Workflow

ode-code integrates into your existing development environment without forcing you to change your tools.

The Loop

The typical workflow follows this pattern:

  1. Context Gathering: You verify which files the AI sees using /ls or /mention.
  2. Prompting: You describe the task using natural language.
  3. Proposal: The AI proposes a shell command or a file edit.
  4. Diff Review: You see a Green/Red diff of the proposed changes.
  5. Confirmation: Press Enter to apply, or describe changes to refine.

Git Integration

ode-code is Git-aware. It can read your `.gitignore` to avoid scanning massive `node_modules` folders, and it can automatically stage or commit changes if you ask it to (e.g., "Fix this bug and commit with message 'fix login'").

Core Capabilities / Commands & Tools

Special Commands & Features

Supercharge your workflow with these native CLI commands.

File Monitoring (/watch)

Real-time monitoring of file changes. If you edit a file in VS Code while ode-code is running, it will immediately notice the change and update its context.

Bug Hunter Mode (/bug)

Enter a specialized debugging workflow. Instead of writing code immediately, the agent will:

  1. Analyze the error stack trace.
  2. Read relevant files.
  3. Propose a hypothesis.
  4. Apply the fix upon confirmation.

Context Marking (/mention)

Similar to Slack or Discord, you can "mention" specific files to force the AI to pay attention to them. E.g., /mention src/utils/auth.js.

Command Execution

ode-code can run terminal commands for you, such as npm test, git status, or ls -la. It reads the output to understand the state of your project.

Experience & Safety / Interactive UX

Interactive User Experience

The CLI is built with Ink (React for CLI), providing a rich, dashboard-like experience inside your terminal.

Key UI Elements:

  • Spinner & Progress Bars: Visual feedback during long-running High Mode tasks so you know the agent hasn't frozen.
  • Syntax Highlighting: All code output is syntax-highlighted for readability.
  • Selectable Menus: When the AI needs you to choose a file or an option, use your Arrow Keys to navigate and Enter to select.

Keyboard Shortcuts

Key Action
Tab Toggle "Deep Thinking" (Reasoning Model)
Ctrl + C Interrupt current generation (Cancel)

Experience & Safety / Safety & Logs

Safety & Reliability

Giving an AI access to your file system requires trust. We've built strict guardrails to prevent accidents.

Permission Tiers

Action Type Default Behavior Configurable?
Read Files Allowed Yes (via .gitignore)
Create Files Auto-allowed Yes
Edit Files Requires Confirmation (Y/n) Yes (Can set to Auto)
Delete Files Strict Confirmation Required No (Always strict)
Shell Commands Requires Confirmation Yes

Audit Logs

Every session creates a local log file in ./.ode-code/logs/. This JSON file contains the full conversation history, prompt usage, and file modifications for auditing purposes.

Experience & Safety / Configuration

Workspace Configuration

You can customize how ode-code behaves in your specific project by creating an ode-code.md file in the root directory.

This file acts as a "System Prompt" for your project. The AI reads this before every request.

Example ode-code.md

ode-code.md
# Project Rules

## Coding Style
- Use Functional Components for React.
- Always use Tailwind CSS for styling.
- Prefer `const` over `let`.
- No `any` types in TypeScript.

## Architecture
- All API calls must go through `@/services/api`.
- Components should be placed in `src/components/domains/{domainName}`.

## Ignored
- Do not read files in `legacy/` folder.

Use the /init command to have ode-code analyze your project and generate a starting `ode-code.md` automatically.
